# InfluxDB Data Schema — Predictive Maintenance System

## Measurement: `sensor_telemetry`

### Tags (Indexed)
- `unit_id` — Engine unit identifier (string)

### Fields (Values)
- `time_cycles` — Operational cycles (integer)
- `operational_setting_1` — Flight condition 1 (float)
- `operational_setting_2` — Flight condition 2 (float)
- `operational_setting_3` — Flight condition 3 (float)
- `sensor_1` through `sensor_21` — 21 sensor readings (float)

### Timestamp
- Auto-generated by InfluxDB at write time

## Example Queries

### Get last 10 readings for unit 5:
```flux
from(bucket: "sensor-data")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "sensor_telemetry")
  |> filter(fn: (r) => r.unit_id == "5")
  |> limit(n: 10)
```

### Average sensor_1 by unit:
```flux
from(bucket: "sensor-data")
  |> range(start: -24h)
  |> filter(fn: (r) => r._measurement == "sensor_telemetry")
  |> filter(fn: (r) => r._field == "sensor_1")
  |> group(columns: ["unit_id"])
  |> mean()
```

## Retention
- Default: 90 days (configurable in bucket settings)
